# 隱私說明

## Replicate API 的隱私情況

### 是的，使用 Replicate API 時，圖片會被上傳到互聯網

**具體流程：**
1. 你的照片和衣服圖片會上傳到 Replicate 的服務器
2. Replicate 使用 AI 模型處理圖片
3. 處理完成後，結果圖片會返回給你
4. **重要：** Replicate 可能會暫時保存你的圖片用於處理

### Replicate 的隱私政策

根據 Replicate 的隱私政策：
- 圖片用於處理請求
- 可能會暫時保存
- 不會用於訓練模型（除非你明確同意）
- 可以刪除你的數據

**詳細信息：** https://replicate.com/privacy

## 如果你不想上傳圖片到互聯網

### 方案 1：使用本地 Canvas 模式（當前已實現）

**優點：**
- ✅ 完全本地處理，不上傳
- ✅ 完全免費
- ✅ 隱私安全

**缺點：**
- ❌ 效果不如 AI 真實（會出現蓋在臉上的問題）
- ❌ 只是簡單的圖片疊加

### 方案 2：自己搭建本地 AI 服務（複雜）

**需要：**
- Python 環境
- GPU（推薦）
- 下載 AI 模型（6-7GB）
- 技術能力

**優點：**
- ✅ 完全本地，不上傳
- ✅ 完全控制

**缺點：**
- ❌ 設置複雜
- ❌ 需要 GPU（CPU 很慢）
- ❌ 維護麻煩

### 方案 3：使用其他隱私保護的服務

一些服務提供：
- 端到端加密
- 自動刪除處理後的圖片
- 不保存原始圖片

但這些通常需要付費且設置複雜。

## 建議

### 如果你關心隱私：

1. **測試階段**：使用 Canvas 模式（雖然效果不完美，但完全本地）

2. **正式使用**：
   - 如果只是個人使用，Replicate 的隱私風險相對較低
   - 可以只上傳不包含敏感信息的照片
   - 處理完成後，Replicate 通常會自動清理

3. **敏感照片**：
   - 不要上傳包含敏感信息的照片
   - 使用 Canvas 模式處理敏感照片

### 隱私保護措施：

1. **只上傳必要的照片**
   - 不要上傳包含個人信息、背景等的照片
   - 可以裁剪或模糊背景

2. **使用測試照片**
   - 先用不重要的照片測試
   - 確認效果後再決定是否使用

3. **定期清理**
   - 處理完成後，刪除上傳的臨時文件
   - 檢查 Replicate 帳號設置
